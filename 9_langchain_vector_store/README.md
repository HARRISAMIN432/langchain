# ChromaDB with LangChain

## Introduction

ChromaDB is an open-source vector database designed to store, index, and retrieve vector embeddings efficiently. It is commonly used in Retrieval-Augmented Generation (RAG), semantic search, recommendation systems, and AI-powered applications.

When integrated with LangChain, ChromaDB acts as the vector store responsible for storing embeddings generated by an embedding model and retrieving the most relevant documents for a user query.

---

# How ChromaDB Works

The overall workflow is:

```
Raw Documents
      │
      ▼
Document Loader
      │
      ▼
Text Splitter
      │
      ▼
Small Chunks
      │
      ▼
Embedding Model
      │
      ▼
Vector Embeddings
      │
      ▼
ChromaDB
      │
      ▼
Similarity Search
      │
      ▼
Retrieved Documents
      │
      ▼
LLM (Gemini/OpenAI/etc.)
      │
      ▼
Final Response
```

# Creating a Chroma Vector Store

```python
from langchain_chroma import Chroma
from langchain_google_genai import GoogleGenerativeAIEmbeddings

embeddings = GoogleGenerativeAIEmbeddings(
    model="gemini-embedding-2-preview",
    google_api_key="YOUR_API_KEY"
)

vector_store = Chroma(
    collection_name="documents",
    embedding_function=embeddings
)
```

---

# Persistent Database

Without specifying `persist_directory`, the database exists only in memory.

```python
vector_store = Chroma(
    collection_name="documents",
    embedding_function=embeddings,
    persist_directory="./chroma_db"
)
```

Directory structure:

```
project/
│
├── app.py
├── chroma_db/
│   ├── chroma.sqlite3
│   ├── index/
│   └── collections/
```

The database is automatically saved.

```python
vector_store = Chroma(
```

# Adding Documents

```python
from langchain_core.documents import Document

documents = [
    Document(
        page_content="Python is a programming language.",
        metadata={"source": "book"}
    ),
    Document(
        page_content="Java is object-oriented.",
        metadata={"source": "website"}
    )
]

vector_store.add_documents(documents)
```

---

# Document Structure

Each document consists of:

```python
Document(
    page_content="Actual text",
    metadata={
        "source":"pdf",
        "page":5,
        "author":"John"
    }
)
```

---

# Metadata

Metadata is stored alongside each embedding.

Example:

```python
metadata={
    "file":"notes.pdf",
    "chapter":"Introduction",
    "page":10,
    "year":2024
}
```

Metadata enables filtering during retrieval.

---

# IDs

Each document has a unique ID.

```python
vector_store.add_documents(
    documents,
    ids=["doc1","doc2"]
)
```

---

# Updating Documents

Delete the old document:

```python
vector_store.delete(ids=["doc1"])
```

Insert the updated version.

---

# Deleting Documents

```python
vector_store.delete(ids=["doc2"])
```

Delete everything:

```python
vector_store.delete_collection()
```

# Similarity Search

Basic search:

```python
results = vector_store.similarity_search(
    "What is Python?",
    k=3
)
```

Returns:

```python
[
    Document(...),
    Document(...),
    Document(...)
]
```

---

# Similarity Search with Score

```python
results = vector_store.similarity_search_with_score(
    "Python",
    k=5
)
```

Returns:

```python
[
    (Document(...),0.12),
    (Document(...),0.18)
]
```

Lower distance generally indicates greater similarity.

---

# Metadata Filtering

```python
results = vector_store.similarity_search(
    "Machine Learning",
    filter={"source":"book"}
)
```

Multiple filters:

```python
filter={
    "source":"book",
    "author":"John"
}
```

---

# Maximum Marginal Relevance (MMR)

MMR balances relevance and diversity.

```python
results = vector_store.max_marginal_relevance_search(
    "Python",
    k=5,
    fetch_k=20
)
```

Parameters:

* `k` → Final results returned.
* `fetch_k` → Initial candidate pool.

# Retrievers

Convert Chroma into a LangChain retriever:


Retrieve documents:

```python
docs = retriever.invoke("Explain Python")
```

# Text Splitting Before Storage

Never embed huge documents directly.

```python
from langchain_text_splitters import RecursiveCharacterTextSplitter

splitter = RecursiveCharacterTextSplitter(
    chunk_size=1000,
    chunk_overlap=200
)

chunks = splitter.split_documents(documents)
```

Then:

```python
vector_store.add_documents(chunks)
```

---

# Embedding Dimensions

Different embedding models produce vectors of different sizes.

Examples:

| Model                      | Dimensions   |
| -------------------------- | ------------ |
| text-embedding-3-small     | 1536         |
| text-embedding-3-large     | 3072         |
| gemini-embedding-2-preview | Configurable |

Never mix embeddings generated from different models in the same collection.


---

# Best Practices

* Split documents before embedding.
* Preserve useful metadata.
* Use persistent storage for production.
* Keep all embeddings in a collection from the same model.
* Use MMR when you want diverse search results.
* Use metadata filters to narrow retrieval.
* Store document IDs for easier updates and deletion.
* Tune `chunk_size`, `chunk_overlap`, and `k` based on your use case.

# Common Methods

| Method                            | Purpose                          |
| --------------------------------- | -------------------------------- |
| `add_documents()`                 | Store LangChain documents        |
| `add_texts()`                     | Store raw text                   |
| `similarity_search()`             | Retrieve similar documents       |
| `similarity_search_with_score()`  | Retrieve with scores             |
| `max_marginal_relevance_search()` | Diverse retrieval                |
| `delete()`                        | Delete by ID                     |
| `delete_collection()`             | Delete the collection            |
| `as_retriever()`                  | Convert to a LangChain retriever |
